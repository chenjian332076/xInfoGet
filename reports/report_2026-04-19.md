### 3. GrandCode：竞技编程现场的 Agentic 强化学习

> 来源: Jiwei Li 等 | arXiv:2604.02721 | 2026-04-03

论文提出 **GrandCode**：通过假设提出、求解、测试生成、总结等多模块协同，并结合针对多阶段 rollout 的 **Agentic GRPO** 处理延迟奖励与严重 off-policy 漂移。作者报告在 2026 年 3 月三场 Codeforces 现场赛中均获第一名并击败全部人类选手（含传奇 Grandmaster）。

**关键观点**: 当评测从静态题库转向「实时赛制」，多智能体 RL 与测试时策略仍能系统性抬升上限，竞技编程作为硬基准的意义会被重写。

🔗 [阅读原文](https://arxiv.org/abs/2604.02721)

### 4. LightThinker++：从推理压缩到显式记忆管理

> 来源: Yuqi Zhu 等 | arXiv:2604.03679 | 2026-04-04

在 LightThinker 动态压缩中间「思考」的基础上，**LightThinker++** 引入显式记忆原语与轨迹合成训练，使模型学会 **何时压缩、何时保留**。摘要报告标准推理峰值 token 降约七成且精度略升；在长程 Agent 场景可稳定超过 80 轮交互， footprint 降 60%–70%，平均增益约 14.8%。

**关键观点**: 长程 Agent 的瓶颈不仅是模型智商，更是 **上下文经济学**——可学习的记忆调度或比单纯「更大窗口」更可持续。

🔗 [阅读原文](https://arxiv.org/abs/2604.03679)

## AI 工程实践

### 6. Project Think：在 Cloudflare 上构建下一代 AI Agent

> 来源: Cloudflare Blog | 2026-04-15

**Project Think** 为 Agents SDK 补充 durable execution（fiber + stash 检查点）、子 Agent（隔离 SQLite + 类型化 RPC）、树状可分叉 **Session API**、**codemode** 下的沙箱 JS 执行与运行时 npm 打包，以及从 Workspace → Dynamic Worker → Browser → 全功能 Sandbox 的 **执行阶梯**。文章认为：Coding Agent 正从本机工具演化为可休眠、可恢复、边际成本接近零的 **互联网级基础设施**。

**关键观点**: 「工具爆炸」时代，让模型写短程序批量操作 API（codemode）比串行 tool call 更省 token；安全应通过 **能力绑定** 默认最小权限实现。

🔗 [阅读原文](https://blog.cloudflare.com/project-think/)

### 7. The next evolution of the Agents SDK

> 来源: OpenAI | 2026-04-15

新版 **Agents SDK** 强调 **模型原生 harness**：可配置记忆、兼容 MCP / skills / `AGENTS.md`、shell 与 apply_patch 等工具，并原生集成 **沙箱执行**。开发者可自带沙箱或使用 Blaxel、Cloudflare、Daytona、E2B、Modal、Runloop、Vercel 等合作方；通过 **Manifest** 描述工作区，并支持从本地到 S3/GCS/Azure/R2 的数据挂载。文章指出将 harness 与执行环境分离，有助于对抗注入与外泄、实现可快照的 durable run，以及多沙箱并行扩展。

**关键观点**: 生产级 Agent = 可靠 harness + 可移植工作区描述 + 与模型行为对齐的执行边界，而不是零散拼装的「又一个框架」。

🔗 [阅读原文](https://openai.com/index/the-next-evolution-of-the-agents-sdk/)

## 趋势总结

1. **智能体基础设施周**：Cloudflare 与 OpenAI 在同周强化 SDK/沙箱/会话模型，说明行业正从「单轮 Chat」转向可休眠、可恢复、可审计的长程 Agent 运行时。
2. **安全与能力绑定发布**：Mythos 类模型展示极端网络能力的同时，Opus 4.7 率先承载可部署的实时网络防护与验证计划，预示「强模型 + 强治理」将成标配叙事。
3. **推理经济学兴起**：LightThinker++ 一类工作聚焦如何在有限上下文下维持深度思考，Agent 场景对 token 峰值与记忆调度的敏感度会越来越高。
4. **评测迁移到「真实工作流」**：从 Codeforces 现场赛到企业文档/金融/合规场景，模型价值越来越依赖多步工具调用与长时间一致性，而非单次 benchmark 分数。
5. **平台与开源并行**：头部闭源 API 与开源权重（如 Gemma、Qwen 系）在编程与 Agent 能力上同步加码，开发者可选择「托管运行时」或「可私有部署栈」。
