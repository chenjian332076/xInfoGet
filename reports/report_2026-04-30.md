## AI 前沿动态

### 1. BioMysteryBench：用「可验证脏数据」测量生信研究型智能

> 来源: Anthropic Science (Brianna 等) | 2026-04-29

文章系统讨论：为何 **聊天式科学题**（MMLU-Pro、GPQA、LAB-Bench 等）不足以刻画 **真实科研流水线**；并介绍自研基准 **BioMysteryBench**——**99 题** 来自专家出题，要求基于 **未加工或弱加工** 的组学/质谱等数据，在 **容器+标准生信工具+可装包** 条件下给出 **客观可核查答案**（如根据 RNA-seq 推断敲除基因、据 metadata 与数据推断病毒感染等）。**76** 题被 **≤5 名专家** 至少一人解出；**23** 题专家集体失败但经过 QC 保留；评测显示多代 **Claude** 在 **人类可解题** 上 **整体准确率与可靠性同步抬升**，且在 **专家也难子集** 上 **Sonnet 4.6 及以上** 仍能解出 **可观比例**，**Mythos Preview** 在最难子集报告约 **30%** 级别表现。文章同时引用同期 **Genentech/Roche 的 CompBioBench（bioRxiv）** 作为 **独立收敛证据**。数据集预览见 Hugging Face：`Anthropic/BioMysteryBench-preview`。

**关键观点**: 科学 AGI 评测正从 **「知识问答」** 迁往 **「带噪真实数据 + 工具闭环 + 可证伪结论」**；同时必须把 ** headline 准确率** 与 **跨多次试验的一致性** 一起读，否则易 **高估** 模型在 **开放脏问题** 上的可用性。

🔗 [阅读原文](https://www.anthropic.com/research/Evaluating-Claude-For-Bioinformatics-With-BioMysteryBench)

### 2. 小米 MiMo-V2.5-Pro：万亿 MoE 开源、百万上下文与长程 Agent 叙事

> 来源: Xiaomi MiMo Team | 页面更新 2026-04-27

官方宣布 **MiMo-V2.5-Pro** 在 **API / AI Studio** 等全量可用且 **价格不变**（模型标签 `mimo-v2.5-pro`）。架构上为 **约 1.02T 总参数、42B 激活** 的 MoE，**混合全局/滑动窗口注意力 + MTP**，原生 **最长 1M token** 上下文，预训练约 **27T token**，并采用 **多阶段后训练 + 多教师 on-policy 蒸馏**。公开案例强调 **极长 tool-call 轨迹**：如 **SysY 编译器 Rust 全流程** 在 **约 672 次工具调用、4.3 小时** 内跑通课程隐藏测 **233/233**；另有 **视频编辑器桌面应用（约 1868 次调用、11.5 小时、8192 行代码）** 与 **模拟电路 ngspice 闭环优化** 等。**ClawEval** 上官方给出 **Pass^3 与每轨迹 token** 的 **效率前沿** 叙事（相对若干闭源前沿模型 **约 40–60% token 节省** 的量级表述）。权重与模型卡托管 **Hugging Face**，并推荐 **SGLang / vLLM** 社区 cookbook 部署。

**关键观点**: 国产开源旗舰正在 **同一叙事战场**（**超长上下文 + MoE + Agent 轨迹成本**）与 **全球前沿闭源** 对齐；工程侧 **推理栈 Day-0 适配** 与 **轨迹经济性** 正在成为 **与裸分同等重要** 的分发变量。

## AI 工程实践

### 1. Symphony：把 Issue Tracker 变成 Codex 的「永不关机」编排层

> 来源: OpenAI Engineering (Alex Kotliarskyi, Victor Zhu, Zach Brock) | 2026-04-27

团队回顾从 **Harness Engineering** 到 **规模化代理会话** 的演进：当每名工程师并行 **3–5** 个 Codex 会话时，**上下文切换** 成为瓶颈。 **Symphony** 定位为 **语言无关** 的长驻服务：**轮询 Linear（首期）**、为 **每个 eligible issue 创建隔离工作区**、以 **`codex app-server` 协议** 拉起会话，并在崩溃/停滞时 **退避重试**；策略写在仓内 **`WORKFLOW.md`**（YAML front matter + Liquid 模板），支持 **热重载**。编排器自述职责边界：**调度 + workspace + 观测**，**票据写回** 由代理经工具完成。官方称内部部分团队 **前三周 merged PR 数量约 +500%**（方向性指标），并引用 **Linear 创始人** 在 X 上关于 **workspace 创建激增** 的公开反馈。仓库 **`openai/symphony`** 以 **`SPEC.md`** 为真值，Elixir 参考实现附列；明确 **不承诺** 作为独立商业产品长期维护，更像 **可复制的合同与参考实现**。

**关键观点**: 当「编码代理」变成 **产线工人**，竞争焦点从 **单次对话智商** 转向 ** ticket 级 SLA、隔离与可观测性**——这与 **Kubernetes/DAG** 思维是同构的。

## 趋势总结

1. **编排层「发票据」**：Symphony 类方案把 **代理实例** 从「工程师脑内会话」挪到 **工单系统**，**人类** 更像 **SRE/产品经理排程**，剩下的是 **治理与测试 harness**。

3. **开源旗舰拼「轨迹经济学」**：MiMo-V2.5-Pro 强调 **同等任务 token 成本**，预示 **Agent 时代定价与路由** 将围绕 **$/有效步** 而非 **$/百万参数** 展开。  
4. **平台安全话语升温**：OpenAI 长文将 **模型规范—监测—人工—执法** 串成 **公共叙事**，与 **企业 agent 合规** 形成 **镜像话题**。  
5. **生态侧即时反馈**：Symphony 发布与 **Linear 侧活跃度** 的公开互动表明 **开发者工具链** 与 **项目管理 SaaS** 在 **agent 浪潮** 中正在 **互相牵引增长**。
