### 1. GPT‑5.5 Instant：默认对话模型换代

> 来源: OpenAI | 2026-05-05

OpenAI 将面向全体用户的 **ChatGPT 默认 Instant 模型**更新为 **GPT‑5.5 Instant**，强调日常回答更凝练、语气更自然，并在内部评测中称相对 **GPT‑5.3 Instant** 在高风险提示上 **幻觉声明减少约 52.5%**，在用户标记的事实错误对话上不准确声明减少约 **37.3%**。同日扩展 **Memory sources**：当回答使用个性化上下文时，用户可查看引用来源并修正或删除。**API** 中该默认轨迹对应 **`chat-latest`**；付费用户可在约三个月内继续选用 GPT‑5.3 Instant。

**关键观点**: 「默认模型」是数十亿人感知 AI 能力的真实界面，事实性与可审计的个性化比榜单分数更能定义产品信任。

## AI 工程实践

### 3. 规模化低延迟语音：OpenAI 的 WebRTC 中继架构

> 来源: OpenAI Engineering（Yi Zhang, William McDonald）| 2026-05-04

文章指出语音交互对 **全球覆盖、快速建连、稳定低抖动 RTT** 的苛刻要求，并解释如何在 **Kubernetes** 环境下规避「每会话独占 UDP 端口」导致的端口耗尽与弹性伸缩难题。方案核心为 **split relay + transceiver**：轻量 **relay** 仅解析 **STUN/ufrag** 做首包路由，**transceiver** 独占 **ICE/DTLS/SRTP** 状态；配合 **Global Relay** 地理入站与 **Cloudflare** 对信令的地理调度，使客户端仍使用标准 WebRTC，而内部获得更小攻击面与更可预测的会话归属。

**关键观点**: 实时多模态 Agent 的瓶颈常常不在模型，而在「如何把连续音频流安全、低成本地接到推理链」——这篇是可复用的边缘架构参考。

🔗 [阅读原文](https://openai.com/index/delivering-low-latency-voice-ai-at-scale/)

## 趋势总结

1. **实时交互栈显性化**：语音 Agent 竞争正下沉到 **WebRTC 路由、全球入站与状态归属**，推理之外的工程复杂度被官方主动披露。  
2. **「默认模型」成为信任战场**：Instant 路线迭代聚焦 **事实性、简洁度与可解释个性化**，反映消费端对幻觉与啰嗦的耐受度已显著降低。  
3. **垂直 Agent 进入模板化交付**：金融场景将 **插件市场 + Office 嵌入式工作流 + 托管 Agent** 捆成 SKU，重现企业软件时代的「配方式实施」。  
4. **资本与交付组织同步变形**：PE 背景合资公司表明 **模型销售**正演化为 **长期驻场集成服务**，竞争维度从 API 调用转向 **管道与关系**。  
5. **开源权重 + 云异步 Agent 并行**：Mistral 路径说明 **中等规模稠密模型** 仍可在编码 Agent 场景取得 **SWE-Bench** 级商用叙事，与闭源前沿形成错位竞争。  
6. **公共话语分裂**：就业冲击的 **高管预言 vs 学术质疑** 同步升温，技术决策者需建立独立的数据监测框架而非跟随口号。
